\chapter*{Control Flow}
\addcontentsline{toc}{chapter}{Control Flow}
In Twin-64, control normally proceeds to the next sequential instruction in memory unless explicitly changed by a branch or interrupted by an exception. From the programmer’s perspective, instructions execute in program order. Internally, however, the hardware may employ techniques such as out-of-order execution. This chapter presents the logical execution model, describing how branching and interruptions alter control flow.
\section*{Unconditional Branches}
\addcontentsline{toc}{section}{Unconditional Branches}
Unconditional branch instructions always redirect control flow and may optionally save the return address in a general-purpose register. The branch target address can be computed in two ways:
\begin{itemize}
\item \textbf{Instruction-relative branches:} The target is obtained by adding an offset to the address of the current instruction.
\item \textbf{Segment-relative branches:} The target is computed relative to the start of the segment (address zero of the segment).
\end{itemize}
Both methods use 32-bit unsigned arithmetic with wrap-around on overflow. For example, if the current instruction resides at \texttt{0xFFFF\_FFFC} and the branch offset is \texttt{0x10}, the calculation is:

\begin{flushleft}
\hspace*{2em}$\texttt{0xFFFF\_FFFC} + \texttt{0x10} = \texttt{0x1\_0000\_000C}$
\end{flushleft}

Since the addition is performed modulo $2^{32}$, the high-order carry is discarded. The effective branch target is therefore \texttt{0x0000\_000C}.
\section*{Conditional Branches}
\addcontentsline{toc}{section}{Conditional Branches}
Conditional branches combine a comparison with an instruction-relative branch. If the specified condition is true, control transfers to the branch target; otherwise, execution continues sequentially. Conditional branches are always instruction-relative.
\section*{Linkage}
\addcontentsline{toc}{section}{Linkage}
Certain branch instructions support procedure calls by saving the return address (the instruction immediately following the branch) into a general-purpose register. The linkage mechanism applies to both intra-segment and inter-segment branches. The saved return address consists of both the segment identifier and the instruction offset.
\section*{Inter-segment Branches}
\addcontentsline{toc}{section}{Inter-segment Branches}
Unconditional inter-segment branches specify both a new segment identifier and an offset within that segment. The \texttt{BE} instruction, for example, computes a global virtual address by combining the segment ID with a base offset, then adding the instruction-encoded displacement.
\section*{Privilege Level Changes}
\addcontentsline{toc}{section}{Privilege Level Changes}
Branch instructions may alter the current privilege level. Execution is only permitted if the current privilege level satisfies the requirements of the code or data being accessed.
Traditionally, privilege changes are handled through traps into the operating system kernel. While secure, this approach incurs significant overhead due to pipeline flushes and handler setup.
Twin-64 provides a more efficient gateway mechanism. An unconditional branch with the \texttt{.G} gateway option may enter a gateway page, raising the privilege level. The processor then executes at the privilege level specified by the gateway page, with the PSW privilege bit updated accordingly.
When returning to code in a lower-privilege page, the CPU automatically lowers the current privilege level. Each instruction fetch checks the privilege level of the code page against the PSW \texttt{P}-bit:
\begin{itemize}
\item If the page requires higher privilege than the PSW indicates, a privilege violation trap occurs.
\item If the page requires lower privilege, execution is automatically demoted.
\end{itemize}
\section*{Traps Associated with Branches}
\addcontentsline{toc}{section}{Traps Associated with Branches}
Branch instructions may trigger traps depending on the PSW state. For example, if the PSW \texttt{T}-bit is set and a branch is taken, a \textit{taken-branch trap} is raised. This feature is primarily intended for debugging.
