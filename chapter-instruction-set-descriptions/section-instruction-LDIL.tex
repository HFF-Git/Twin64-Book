\pagebreak
\section*{LDIL — Load Immediate Left}
\addcontentsline{toc}{section}{LDIL — Load Immediate Left}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Syntax}
\texttt{LDIL[.L/.S/.U] RegR, val}
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Format}
The \texttt{LDIL} instruction uses the immediate-20 instruction format. \\

\begin{flushleft}
\drawInstrFormat
    {0,1,3,5,6,16}
    {\OpCodeGrpALU,\OpCodeLDIL, RegR, mode, val}
\end{flushleft}
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Description}
The \texttt{LDIL} instruction deposits an immediate value into general register \texttt{RegR}.  
The optional suffix determines how the immediate value is positioned within the destination register. If no suffix is specified, the default is \texttt{.L} mode (mode value = 1).

\begin{itemize}
    \item \texttt{.L} (mode = 1) — Deposits 20 bits of \texttt{val} into bits [31..12] of \texttt{RegR}. RegR is cleared before the operation. 
    \item \texttt{.S} (mode = 2) — Deposits 20 bits of \texttt{val} into bits [51..32] of \texttt{RegR}. RegR is not cleared before the operation.
    \item \texttt{.U} (mode = 3) — Deposits 12 bits of \texttt{val} into bits [63..52] of \texttt{RegR}. RegR is not cleared before the operation.
\end{itemize}

A mode value of zero is reserved and must not be used.
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescOpEntry}{Operation}
\begin{lstlisting}[style=iPageOpStyle]
RegR <- ((val & 0xFFFFF) << 12)                   // .L mode (1)
RegR <- deposit( RegR, (val & 0xFFFFF), 32, 20 ); // .S mode (2)
RegR <- deposit( RegR, (val & 0xFFF), 52, 12 );   // .U mode (3)
\end{lstlisting}
\end{iPageDescOpEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Exceptions}
None.
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Notes}
The \texttt{LDIL} instruction is used to construct large values in a register.  
The \texttt{.L} variant clears the destination register before depositing the immediate value.  
This form is intended for building larger offsets for indexed addressing modes, where an addressing instruction adds its own 12-bit offset in the low bits. Together, these operations allow the construction of any 32-bit offset.

In contrast, the \texttt{.S} and \texttt{.U} variants deposit their immediate fields without clearing the register, preserving the remaining bits. Using a combination of these modes, a full 64-bit value can be assembled in two to four instructions.
\end{iPageDescEntry}
