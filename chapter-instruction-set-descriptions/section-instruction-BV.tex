\section*{BV - Branch Vectored}
\addcontentsline{toc}{section}{BV — Branch Vectored}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Syntax}
\texttt{BV RegB [, RegR, RegX ]}
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Format}
The \texttt{BV} instruction uses the register instruction format. \\

\begin{flushleft}
\drawInstrFormat
    {0,1,3,5,6.5,8.5,9.5,11.5,16}
    {\OpCodeGrpBR,\OpCodeBV, RegR, 1, RegB, 0, RegX, 0}
\end{flushleft}
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Description}
The \texttt{BV} instruction performs an unconditional branch to the address stored in \texttt{RegB}.  

The target is interpreted as an instruction address within the current code segment. If code translation is disabled, the target address is the absolute physical address.  

Before branching, the next instruction address (\texttt{IA + 4}) is stored in \texttt{RegR}.  

Because \texttt{BV} is a segment base relative branch, branching to a page with a different privilege level is possible.  
Branching from a lower privilege level to a higher one triggers an instruction protection trap.  
Branching from a higher privilege to a lower privilege level updates the privilege level in the processor status register.  
If no change is required, the privilege level remains unchanged.  

Indexed formats may cause a memory-reference–related trap. ( ??? )
\end{iPageDescEntry}

\begin{iPageDescOpEntry}{Operation}
