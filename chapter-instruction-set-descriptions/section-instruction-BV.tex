\section*{BV - Branch Vectored}
\addcontentsline{toc}{section}{BV — Branch Vectored}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Syntax}
\texttt{BV RegB [, RegR, RegX ]}
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Format}
The \texttt{BV} instruction uses the register instruction format. \\

\begin{flushleft}
\drawInstrFormat
    {0,1,3,5,6.5,8.5,9.5,11.5,16}
    {\OpCodeGrpBR,\OpCodeBV, RegR, 1, RegB, 0, RegX, 0}
\end{flushleft}
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Description}
The \texttt{BV} instruction performs an unconditional segment relative branch. The content of \texttt{RegX} is sign extended and shifted by two and added to \texttt{RegB}, forming the target offset in the current segment. The return link is store in \texttt{RegR}.  

Because \texttt{BV} is a segment base relative branch, branching to a page with a different privilege level is possible. Branching from a lower privilege level to a higher one triggers an instruction protection trap. Branching from a higher privilege to a lower privilege level updates the privilege level in the processor status register. If no change is required, the privilege level remains unchanged.  
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescOpEntry}{Operation}
\begin{lstlisting}[style=iPageOpStyle]
RegR <- addOfs32( PSW.[IA], 4 );
PSW.[IA] <- addOfs32( RegB, RegX << 2 );
\end{lstlisting}
\end{iPageDescOpEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Exceptions}
Taken Branch Trap\\
Instruction TLB Miss Trap
\end{iPageDescEntry}

%--------------------------------------------------------------------------------
\begin{iPageDescEntry}{Notes}
None.
\end{iPageDescEntry}
