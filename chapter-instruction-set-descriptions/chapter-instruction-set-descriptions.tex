\chapter*{Instruction Set Descriptions}
\addcontentsline{toc}{chapter}{Instruction Set Descriptions}

This chapter provides a detailed description of the Twin-64 instruction set. Each instruction entry includes:

\begin{itemize}
    \item The instruction word layout.
    \item A concise description of the instruction.
    \item A high-level pseudo code representation of its operation.
\end{itemize}

Instruction operations are expressed using C-style pseudo code. The routines and conventions used are listed in the appendix. Registers are referenced by class and index:

\begin{itemize}
    \item \texttt{GR[5]} — general register 5
    \item \texttt{CR[1]} — control register 1
    \item Some registers have dedicated names, e.g., the shift amount control register is \texttt{SAR}.
\end{itemize}

Subfields of an instruction are accessed using a dot and the field name in square brackets. For example:

\begin{itemize}
    \item The interrupt enable bit in the PSW register: \texttt{PSW.[I]}.
\end{itemize}

Instruction options are specified in assembler syntax by a dot followed by one or more characters, each representing a defined option. For example:

\begin{itemize}
    \item \texttt{AND.N ...} — the \texttt{AND} instruction with the negate result option.
    \item The order of option characters does not matter; all defined options are listed in the relevant character set.
\end{itemize}

Fields marked as reserved are intended for future enhancements and should always be set to zero. Any other value in a reserved field results in undefined behavior.
